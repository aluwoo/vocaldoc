# 声乐课堂嗓音档案管理系统 - 需求规格说明书

## 1. 项目概述

### 1.1 项目背景
随着声乐教育的发展，对学生嗓音状态的系统化记录和分析变得尤为重要。传统的手工记录方式效率低下，难以进行历史对比和趋势分析。本系统旨在为声乐教育机构提供一套完整的嗓音档案数字化管理解决方案。

### 1.2 项目目标
- 建立学生嗓音档案的数字化管理平台
- 支持多端录音（手机、专业设备、音频文件上传）
- 提供嗓音特征自动分析功能
- 支持历史数据对比和趋势分析
- 实现教师与学生的便捷交互

### 1.3 系统架构
- **架构模式**: B/S（Browser-Server）架构
- **部署方式**: 云服务器部署
- **移动端适配**: 响应式设计，兼容iOS/Android浏览器

| 查看自己档案 | ✓ | ✓ | ✓ |
| 嗓音分析 | ✓ | ✓ | ✓ |

### 3.2 嗓音档案模块

#### 3.2.1 档案类型
嗓音档案分为三种类型，分别用于不同场景：

| 类型 | 用途 | 时长 | 自动分析 | 教师点评 |
|------|------|------|----------|----------|
| **元音录音** | 音色/音准评估 | 3-10秒 | ✅ 完整分析 | ✅ 可选点评 |
| **声时长测试** | 气息支撑评估 | 最长30秒 | ✅ 时长+气息分析 | ✅ 可选点评 |
| **完整曲目** | 演唱作品存储 | 1-10分钟 | ❌ 仅存储 | ✅ 详细点评 |

#### 3.2.2 元音录音规范
- **采样元音**: a / i / o / u / e（各单独录制）
- **录音引导**: 屏幕提示当前元音，录制3-5秒
- **自动分析**: 对每个元音进行完整嗓音参数分析
- **对比基准**: 可与标准元音声学参数对比

#### 3.2.3 声时长测试规范
- **测试内容**: 最长持续发声时间 (MPT - Maximum Phonation Time)
- **测试元音**: 通常使用 /a/ 或 /i/ 元音
- **测试方式**: 深吸一口气后持续发声，记录时长
- **评估指标**:
  - 实际持续时长（秒）
  - 气息支撑评分（根据年龄/性别参考值）
  - 气息稳定性（音量衰减曲线）
- **参考标准**:
  - 成年男性: ≥25秒为优秀
  - 成年女性: ≥20秒为优秀
  - 儿童: ≥15秒为优秀

#### 3.2.4 档案基本信息
- **学生信息**: 姓名、年龄、性别、联系方式
- **档案信息**: 档案编号、创建时间、所属教师
- **录音信息**: 录音日期、录音时长、录音设备类型

#### 3.2.4 录音功能
- **手机实时录音**: 使用手机麦克风实时录制
- **专业设备录音**: 支持外接专业麦克风设备
- **音频文件上传**: 支持上传已有音频文件（mp3、wav、m4a格式）
- **录音回放**: 录制完成后可预览回放
- **元音引导模式**: 屏幕显示当前应录制元音 (a/i/o/u/e)
- **声时长引导模式**: 屏幕显示倒计时+波形，持续发声直到无力继续

#### 3.2.5 嗓音分析功能
> **注意**: 自动嗓音分析仅针对元音录音和声时长测试，完整曲目仅做存储和播放

- **元音分析范围**:
  - 单个元音 (a/i/o/u/e) 独立分析
  - 多元音综合评估
- **声时长分析**:
  - 持续发声时长（秒）
  - 气息支撑评分
  - 音量稳定性（衰减曲线）
- **基础参数分析**:
  - 音高检测（基频F0）
  - 声时长（持续时间）
  - 音量/响度
- **高级参数分析**:
  - 歌唱共振峰（Formants F1-F4）
  - 颤音分析
  - 气息支撑评估
  - 音质评分
- **微扰分析**:
  - **频率微扰 (Jitter)**:
    - Jitter (local) - 局部频率微扰
    - Jitter (rap) - 相对平均微扰
    - Jitter (ppq5) - 周期微扰商
    - Jitter (ddp) - 方向性微扰差
  - **振幅微扰 (Shimmer)**:
    - Shimmer (local) - 局部振幅微扰
    - Shimmer (local, dB) - 分贝振幅微扰
    - Shimmer (apq3) / (apq5) / (apq11) - 振幅微扰商
  - **谐噪比 (HNR)** - 谐波与噪声比
  - **倒谱峰显性 (CPP)** - 嗓音质量评估
- **AI智能分析**:
  - 音色特征提取
  - 与标准音色对比
  - 个性化改进建议生成
- **可视化展示**:
  - 频谱图
  - 音高曲线
  - 音量包络
  - 微扰趋势图

#### 3.2.6 教师评价
> **元音录音**: 可选点评  |  **完整曲目**: 详细点评（必填）

- **文字评语**: 教师填写详细评价
- **星级评分**: 1-5星综合评分
- **标签评价**: 预设标签（气息稳定、音色甜美等）

### 5.4 语音分析方案 (基于Parselmouth)

#### 5.4.1 核心库介绍

**Parselmouth** 是Praat的Python接口，提供完整的语音分析功能：
- 官网: https://parselmouth.readthedocs.io/
- 安装: `pip install praat-parselmouth`

#### 5.4.2 分析参数配置

```python
import parselmouth

# 基础参数
SAMPLING_RATE = 44100        # 采样率
TIME_STEP = 0.01             # 时间步长 (10ms)
PITCH_FLOOR = 75             # 最低音高 (Hz) - 女低音
PITCH_CEILING = 800          # 最高音高 (Hz) - 男高音

# 共振峰参数 (根据声音类型调整)
FORMANT_CEILING = 5000       # 成人女声
FORMANT_CEILING = 5500       # 成人男声
FORMANT_CEILING = 8000       # 童声

# 微扰分析参数 ( Praat标准)
JITTER_TIME_STEP = 0.0001    # 起点
JITTER_WINDOW_LENGTH = 0.02   # 窗口长度
JITTER_FILTER = 1.3           # 滤波器
```

#### 5.4.3 嗓音参数指标

| 指标 | 方法 | 说明 | 正常参考范围 |
|------|------|------|--------------|
| **音高 (F0)** | `to_pitch()` | 基频，感知的主要音高 | 80-400 Hz |
| 音高范围 | `pitch.get_range()` | 最高/最低音差 | - |
| 音量 | `to_intensity()` | 响度/dB | 30-90 dB |
| **共振峰 (F1-F4)** | `to_formant_burg()` | 歌唱共鸣频率 | 见下表 |
| **频率微扰 (Jitter)** | `Get jitter (local)` | 频率稳定性 | <1.04% |
| Jitter (rap) | `Get jitter (rap)` | 相对平均微扰 | <0.8% |
| Jitter (ppq5) | `Get jitter (ppq5)` | 周期微扰商 | <1.0% |
| **振幅微扰 (Shimmer)** | `Get shimmer (local)` | 振幅稳定性 | <3.81% |
| Shimmer (dB) | `Get shimmer (local, dB)` | 分贝振幅微扰 | <0.5 dB |
| Shimmer (apq3) | `Get shimmer (apq3)` | 振幅微扰商 | <3% |
| **谐噪比 (HNR)** | `Get harmonicity (cc)` | 谐波/噪声比 | >10 dB |
| **倒谱峰显性 (CPP)** | `Get CPP` | 嗓音质量指标 | >12 dB (正常) |

#### 5.4.3.1 共振峰参考值

| 元音 | F1 (Hz) | F2 (Hz) | F3 (Hz) |
|------|---------|---------|---------|
| /a/ | 800-1000 | 700-1200 | 2500-2800 |
| /e/ | 300-500 | 2000-2500 | 2800-3000 |
| /i/ | 250-350 | 2000-2800 | 3000-3200 |
| /o/ | 400-600 | 600-1000 | 2500-2800 |
| /u/ | 250-350 | 600-1000 | 2500-2800 |

#### 5.4.4 分析服务架构

```python
# vocal-analysis/analysis/service.py
from parselmouth import Sound
import numpy as np

class VoiceAnalysisService:
    
    def analyze_pitch(self, audio_path: str) -> dict:
        """音高分析"""
        sound = Sound(audio_path)
        pitch = sound.to_pitch(
            time_step=0.01,
            pitch_floor=75,
            pitch_ceiling=800
        )
        return {
            'mean_f0': pitch.get_mean_frequency(),
            'min_f0': pitch.get_minimum(),
            'max_f0': pitch.get_maximum(),
            'f0_contour': pitch.selected_array['frequency'].tolist()
        }
    
    def analyze_formants(self, audio_path: str, voice_type: str) -> dict:
        """共振峰分析"""
        ceiling = 8000 if voice_type == 'child' else 5500
        sound = Sound(audio_path)
        formants = sound.to_formant_burg(
            time_step=0.01,
            maximum_formant=ceiling
        )
        # 提取F1-F4
        return {...}
    
    def analyze_intensity(self, audio_path: str) -> dict:
        """音量分析"""
        sound = Sound(audio_path)
        intensity = sound.to_intensity(time_step=0.01)
        return {
            'mean_dB': intensity.get_mean(),
            'max_dB': intensity.get_maximum()
        }
```
